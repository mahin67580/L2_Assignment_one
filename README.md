
১. Interface আর Type - দুই ভাইয়ের গল্প

TypeScript-এ "interface" আর "type" দুজনেই একই কাজ করে - তারা বলে দেয় ডেটার গঠন কেমন হবে। কিন্তু তাদের কাজের স্টাইল একটু আলাদা।

Interface যেমন একটি সরকারি দলিল:
- একই দলিলে আপনি পরবর্তীতে নতুন শর্ত যোগ করতে পারেন
- এটি ক্লাসের সাথে সহজে কাজ করতে পারে
- বড় প্রজেক্টে বেশি ব্যবহৃত হয়

উদাহরণ:
typescript
interface মানুষ {
    নাম: string;
    বয়স: number;
}

// পরে আরও তথ্য যোগ করা যায়
interface মানুষ {
    ঠিকানা: string;
}

// এখন মানুষের মধ্যে নাম, বয়স, ঠিকানা - তিনটাই আছে


Type যেমন একটি চুক্তিপত্র:
- একবার লিখলে পরিবর্তন করা যায় না
- জটিল কন্ডিশনের জন্য ভালো কাজ করে
- ছোট প্রজেক্টে সুবিধাজনক

উদাহরণ:
typescript
type স্ট্যাটাস = 'সক্রিয়' | 'নিষ্ক্রিয়'; // শুধু এই দুইটা ভ্যালুই নেবে

type কর্মচারী = {
    নাম: string;
    পদবী: string;
    স্ট্যাটাস: স্ট্যাটাস;
};


সহজ কথায়:
- বড় এবং জটিল প্রজেক্টে "interface" ব্যবহার করবেন
- ছোট এবং সিম্পল প্রজেক্টে "type" ব্যবহার করবেন
- দুজনেই ভালো, শুধু কাজের ধরণ আলাদা



২. keyof - গুপ্তচরের চাবি

"keyof" হলো TypeScript-এর একটি গুপ্তচর যে কোনো অবজেক্টের সব চাবির (keys) তালিকা তৈরি করে রাখে।

ভাবুন আপনি একটি গাড়ির মালিক:
typescript
interface গাড়ি {
    ব্র্যান্ড: string;
    মডেল: string;
    বছর: number;
    রং: string;
}


keyof দিয়ে চাবির তালিকা তৈরি:
typescript
type গাড়ির_চাবি = keyof গাড়ি;
// এখন গাড়ির_চাবি = "ব্র্যান্ড" | "মডেল" | "বছর" | "রং"


রিয়েল লাইফ উদাহরণ - স্মার্ট চাবিকাঠি:
typescript
function গাড়ির_তথ্য_নাও<T, K extends keyof T>(গাড়ি: T, চাবি: K): T[K] {
    return গাড়ি[ চাবি ];
}

const আমার_গাড়ি: গাড়ি = {
    ব্র্যান্ড: "টয়োটা",
    মডেল: "ক্যামরি",
    বছর: ২০২৩,
    রং: "সিলভার"
};

// সঠিক ব্যবহার
const গাড়ির_রং = গাড়ির_তথ্য_নাও(আমার_গাড়ি, "রং"); // কাজ করবে

// ভুল ব্যবহার - TypeScript সাথে সাথে ধরে ফেলবে
const গাড়ির_দাম = গাড়ির_তথ্য_নাও(আমার_গাড়ি, "দাম"); 
// এরর: "দাম" নামে কোনো প্রপার্টি গাড়িতে নেই!


keyof-এর সুবিধা:
- ✅ভুল থেকে বাঁচায়: ভুল প্রপার্টি নাম লিখলে সাথে সাথে বলে দেবে
- ✅অটো কম্প্লিট: IDE আপনাকে প্রপার্টি নাম সাজেস্ট করবে
- ✅ টাইপ সেফ: রানটাইমে এরর হওয়ার সম্ভাবনা কমে
